# Playing Card Icon Design Brief

This brief applies to the Skill Card icon artwork used on the public website and any future printable card-deck prototypes.

The icons should work as functional game symbols first and decorative artwork second. They need to be instantly recognisable, visually consistent and readable at small card size.

## Core Quality Attributes

### 1. Immediate Recognition

Each icon should communicate its meaning within 1-2 seconds.

Use familiar visual metaphors where possible. Avoid cleverness that only makes sense after explanation. Every icon should have a clear subject, action or category.

### 2. Legibility At Final Card Size

Icons must be designed for the actual size they appear on the card, not only for enlarged screen previews.

They should avoid thin lines, tiny details and subtle textures. Use bold shapes, generous gaps and clear contrast. They should still work at arm's length, on a phone screen or in imperfect lighting.

### 3. Strong Silhouette

The outer shape should be distinctive before internal detail is noticed.

An icon should remain recognisable in black and white, as a flat silhouette, without relying on internal texture or colour. If two icons have similar outlines, they are not ready.

### 4. Consistency As A Set

The icons should feel like one deck.

Keep consistent line weight, corner radius, perspective, fill style, shadow style and visual density. Do not mix a detailed miniature scene with a simple pictogram.

### 5. Distinctiveness Between Icons

Each icon must be easy to distinguish from the others during quick scanning.

Check for similar shapes, colours, internal patterns and meanings. Players should not have to study the card to tell two powers apart.

### 6. Print Reliability

The icon should survive real-world card printing.

Use thick enough lines, clear gaps between shapes, no important details near trim edges and no reliance on ultra-subtle colour differences. Fine detail that looks good on screen can disappear in print.

### 7. Colour Supports Meaning

Colour can reinforce meaning, but the icon should not depend on colour alone.

Shape is the primary identifier. Icons should still make sense in greyscale and should not rely only on red versus green.

### 8. Clear Visual Hierarchy

The icon should sit comfortably in its assigned card-art space.

It should be noticeable but not overpower the card title or rules text. Leave enough padding around the icon so it reads as a category marker, not a decorative border.

### 9. Thematic Fit

The icons should belong to The Invention Club's world: tactile, workshop-like, practical and child-friendly without being babyish.

Useful visual metaphors include wheels, buttons, sensors, trusses, batteries, tools, signals, arrows, test marks and simple prototype shapes. Avoid generic clip art, SaaS-style line icons and dark coding imagery.

### 10. Low Cognitive Load

Each icon should represent one idea.

Avoid combining too many concepts. A good icon system speeds the game up rather than making children decode symbols.

## Practical Acceptance Tests

An icon is good enough when:

- it is understandable at final card size;
- it is recognisable in black and white;
- it is not easily confused with another icon in the set;
- it still works when printed slightly darker, lighter or smaller than expected;
- a new player can make a reasonable guess at its meaning;
- it looks like it belongs with the rest of the deck.

## Useful Design Rule

Start with the simplest version that communicates the idea, then add only the detail that improves recognition, theme or delight. Anything else is decoration and should probably be removed.

## Current Skill Icon Subjects

- `Make it move`: bold wheel or wheel-and-motion-arrow symbol.
- `Make it obey`: large push button or control switch symbol.
- `Make it notice`: sensor-eye or sensing module symbol.
- `Make it strong`: triangular truss or braced-frame symbol.
- `Make it run safely`: battery plus protection/shield symbol.
- `Make it real`: folded cardboard, ruler or cutting/fabrication symbol.
- `Make it decide`: branching path, decision diamond or rule-flow symbol.
- `Make it signal`: antenna, radio waves or message-signal symbol.
- `Make it work again`: magnifying glass plus repair/test symbol.
- `Make it better`: circular improvement arrow plus upgraded prototype symbol.
